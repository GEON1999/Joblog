# 공고 스냅샷은 수동 붙여넣기로 수집

공고 원문 보존은 이 앱의 핵심 가치지만, URL 자동 크롤링이 아니라 본문 수동 붙여넣기 + 원본 URL 메타 기록으로 수집한다. 채용 플랫폼들은 로그인월·봇 차단·SPA 동적 렌더링 때문에 크롤링 실패율이 높고, 사이트별 파서 유지보수는 이 프로젝트 범위를 넘는다. 수동 붙여넣기는 UX 비용이 있지만 저장 성공이 보장되고, 사용자가 저장 내용을 눈으로 확인한 상태로 남는다.

## Considered Options

- **URL 입력 시 서버 크롤링** — UX는 좋지만 실패가 잦으면 "저장했다고 믿었는데 비어 있는" 최악의 결과를 낳는다. 아카이브 도구에서 저장 신뢰성은 편의성보다 우선한다.

## Consequences

- 스냅샷 수정은 허용하되, "지원 시점 원문 보존"이 목적임을 UI에 명시한다.
- 크롤링은 2차 개선 후보로 남겨두며, 도입하더라도 수동 붙여넣기의 대체가 아니라 프리필(pre-fill) 보조로 한정한다.
