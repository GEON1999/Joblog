# Next.js 채택 — SSR이 아니라 서버 통합이 목적

이 앱은 전 화면이 인증 뒤의 개인화 화면이라 Next.js의 대표 강점인 SEO/공개 페이지 SSR은 가치가 없다. 그럼에도 React SPA + 별도 백엔드 대신 Next.js(App Router)를 선택한 이유는 서버 기능 — 이력서 파일 업로드, ICS 피드 생성, 이메일 알림 스케줄러, 인증 미들웨어 — 을 별도 백엔드 레포·배포 없이 한 프로젝트 안에서 해결하기 위해서다.

## Considered Options

- **React SPA(Vite) + 경량 백엔드 분리** — 개인화 앱의 성격에는 더 정직한 구조지만, 싱글유저 도구에 레포·배포·관리 포인트가 2배가 되는 비용을 정당화하지 못한다.

## Consequences

- 렌더링 전략은 SSR 최적화에 집착하지 않는다. 화면은 클라이언트 중심으로, 서버는 데이터 접근·파일·스케줄러 역할에 집중한다.
