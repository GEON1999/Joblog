# ICS 피드는 세션이 아니라 비밀 토큰으로 인증한다

다음 액션을 캘린더에서 보기 위한 ICS 피드(`/api/ics`)는 Supabase 세션 쿠키가 아니라 URL의 비밀 토큰(`?token=…`)으로 인증한다. 구글/애플 캘린더 같은 구독 클라이언트는 서버가 주기적으로 폴링하는 방식이라 우리 로그인 쿠키를 실을 수 없고, OAuth 흐름도 태울 수 없기 때문이다. 토큰은 환경변수(`ICS_FEED_TOKEN`)로 두고, 일치하지 않으면 404를 반환한다.

## Considered Options

- **세션 쿠키 인증** — 앱의 다른 모든 경로와 일관되지만 캘린더 클라이언트가 쿠키를 보내지 못해 피드가 항상 401이 된다. 채택 불가.
- **인증 없이 공개** — 지원 일정은 민감 정보이므로 배포된 URL에 무방비로 노출할 수 없다.

## Consequences

- `/api/ics`는 proxy(미들웨어)의 보호 매처에서 제외한다 — 제외하지 않으면 미인증으로 간주되어 `/login`으로 리다이렉트되고 캘린더가 피드를 못 읽는다. 대신 라우트 핸들러 안에서 토큰을 직접 검증한다.
- 토큰은 URL에 노출되므로 세션만큼 강한 비밀은 아니다. 유출 시 무효화는 `ICS_FEED_TOKEN` 값을 교체하는 것으로 처리한다(구독 URL 재발급). 싱글유저 도구의 위협 모델에서 감수하는 트레이드오프다.
- 피드는 읽기 전용이며 미완료 액션만 노출한다. 쓰기 경로는 없다.
